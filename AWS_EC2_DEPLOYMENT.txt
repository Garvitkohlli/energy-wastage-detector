================================================================================
                    AWS EC2 DEPLOYMENT GUIDE
================================================================================

STEP 1: LAUNCH EC2 INSTANCE
----------------------------

1. Go to AWS Console → EC2 → Launch Instance

2. Choose AMI:
   - Ubuntu Server 22.04 LTS (recommended)
   - Or Amazon Linux 2023

3. Choose Instance Type:
   - t2.micro (Free tier - 1GB RAM, 1 vCPU)
   - t2.small (2GB RAM, 1 vCPU) - recommended for production
   - t3.small (2GB RAM, 2 vCPU) - better performance

4. Configure Security Group:
   Add these inbound rules:
   - SSH (22) - Your IP
   - HTTP (80) - 0.0.0.0/0
   - Custom TCP (5000) - 0.0.0.0/0

5. Create/Select Key Pair:
   - Download .pem file
   - Save it securely

6. Launch Instance


STEP 2: CONNECT TO EC2
-----------------------

Windows (using PowerShell):
> ssh -i "your-key.pem" ubuntu@your-ec2-public-ip

Linux/Mac:
$ chmod 400 your-key.pem
$ ssh -i "your-key.pem" ubuntu@your-ec2-public-ip


STEP 3: INSTALL DEPENDENCIES
-----------------------------

# Update system
sudo apt-get update
sudo apt-get upgrade -y

# Install Git
sudo apt-get install -y git

# Install Docker
sudo apt-get install -y docker.io
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker ubuntu

# Install Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# Log out and log back in for Docker group to take effect
exit


STEP 4: UPLOAD YOUR CODE
-------------------------

Option A: Using Git (recommended)
1. Push your code to GitHub
2. On EC2:
   git clone https://github.com/your-username/your-repo.git
   cd your-repo

Option B: Using SCP (from your local machine)
Windows PowerShell:
> scp -i "your-key.pem" -r C:\path\to\project ubuntu@your-ec2-ip:~/

Linux/Mac:
$ scp -i "your-key.pem" -r /path/to/project ubuntu@your-ec2-ip:~/


STEP 5: DEPLOY APPLICATION
---------------------------

# Navigate to project directory
cd ~/your-project-folder

# Make deploy script executable
chmod +x deploy-ec2.sh

# Run deployment script
./deploy-ec2.sh

# Or manually:
docker-compose up -d --build


STEP 6: VERIFY DEPLOYMENT
--------------------------

# Check if containers are running
docker-compose ps

# View logs
docker-compose logs -f

# Test the application
curl http://localhost:5000

# Access from browser
http://your-ec2-public-ip:5000


STEP 7: SETUP NGINX (OPTIONAL - FOR PRODUCTION)
------------------------------------------------

# Install Nginx
sudo apt-get install -y nginx

# Create Nginx config
sudo nano /etc/nginx/sites-available/energy-monitor

# Add this configuration:
server {
    listen 80;
    server_name your-ec2-public-ip;

    location / {
        proxy_pass http://localhost:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Enable the site
sudo ln -s /etc/nginx/sites-available/energy-monitor /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

# Now access via: http://your-ec2-public-ip (port 80)


STEP 8: SETUP SSL (OPTIONAL - FOR HTTPS)
-----------------------------------------

# Install Certbot
sudo apt-get install -y certbot python3-certbot-nginx

# Get SSL certificate (requires domain name)
sudo certbot --nginx -d your-domain.com

# Auto-renewal is configured automatically

TROUBLESHOOTING
---------------

Issue: Cannot connect to EC2
Solution: Check Security Group allows port 5000 and your IP for SSH

Issue: Docker permission denied
Solution: sudo usermod -aG docker ubuntu, then logout and login

Issue: Application not starting
Solution: Check logs with: docker-compose logs

Issue: Out of memory
Solution: Upgrade to t2.small or t3.small instance

Issue: Database not persisting
Solution: Check ./data directory exists and has correct permissions


COST ESTIMATION
---------------

t2.micro (Free tier eligible):
- First 750 hours/month free for 12 months
- After: ~$8.50/month

t2.small:
- ~$17/month

t3.small:
- ~$15/month

Data transfer:
- First 100GB/month free
- After: $0.09/GB


SECURITY BEST PRACTICES
------------------------

1. Change default SECRET_KEY in .env file
2. Restrict SSH access to your IP only
3. Use SSL/HTTPS in production
4. Keep system updated: sudo apt-get update && sudo apt-get upgrade
5. Use strong passwords for user accounts
6. Enable AWS CloudWatch for monitoring
7. Setup automated backups


BACKUP STRATEGY
----------------

# Backup database
docker-compose exec web cp /app/energy_monitor.db /app/data/backup.db

# Download backup to local machine
scp -i "your-key.pem" ubuntu@your-ec2-ip:~/project/data/backup.db ./

# Automated backup script (add to crontab)
0 2 * * * docker-compose exec web cp /app/energy_monitor.db /app/data/backup-$(date +\%Y\%m\%d).db


MONITORING
----------

# Setup CloudWatch (AWS Console)
1. Go to CloudWatch
2. Create alarm for CPU > 80%
3. Create alarm for Memory > 80%
4. Setup SNS notifications

# Application monitoring
docker-compose logs -f | grep ERROR


AUTO-START ON REBOOT
---------------------

# Create systemd service
sudo nano /etc/systemd/system/energy-monitor.service

# Add:
[Unit]
Description=Energy Monitor
Requires=docker.service
After=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/home/ubuntu/your-project
ExecStart=/usr/local/bin/docker-compose up -d
ExecStop=/usr/local/bin/docker-compose down

[Install]
WantedBy=multi-user.target

# Enable service
sudo systemctl enable energy-monitor
sudo systemctl start energy-monitor


SCALING
-------

For multiple users or high traffic:
1. Upgrade to larger instance (t3.medium or t3.large)
2. Use RDS for database instead of SQLite
3. Setup load balancer (AWS ELB)
4. Use Auto Scaling Group
5. Consider ECS or EKS for container orchestration
